
# Research Agent 🤖

Research Agent — це інтелектуальний AI-агент, створений на базі ReAct архітектури (Reasoning and Acting) з використанням **LangGraph** та **LangChain**. Агент здатен приймати запити, самостійно вирішувати, які інструменти використовувати, шукати інформацію в інтернеті, аналізувати її та генерувати структуровані звіти у форматі Markdown.

## 🌟 Основні можливості

- **Автономний пошук інформації**: Агент використовує пошукову систему DuckDuckGo для знаходження релевантних посилань.
- **Глибокий аналіз сторінок**: Аналізує повні тексти статей за допомогою `trafilatura`, очищуючи їх від реклами та непотрібних елементів.
- **Context Engineering**: Інтелектуальне обрізання занадто довгих текстів, щоб не переповнювати контекстне вікно моделі.
- **Генерація структурованих звітів**: Уся зібрана інформація оформлюється і самостійно записується агентом у готові `.md` звіти.
- **Підтримка багатоекранних діалогів**: Використовує `MemorySaver` з LangGraph для збереження контексту протягом усієї сесії.

---

## 🛠 Технологічний стек

- **LLM**: Google Gemini (`gemini-2.5-flash`), ініціалізована через `ChatGoogleGenerativeAI` з низькою температурою для кращої фактологічності.
- **Фреймворк для агента**: `langgraph.prebuilt.create_react_agent`
- **Інструменти (Tools)**:
  - `web_search`: знаходження списку джерел.
  - `read_url`: детальне вичитування інформації за посиланням.
  - `write_report`: запис результатів у директорію `example_output/`.
  - `get_current_time`: отримання поточного часу для розуміння часового контексту.
  - `read_local_file`: читання вже збережених локальних файлів (звітів).
- **Конфігурація**: Pydantic `BaseSettings` (`config.py`).

## 🚀 Встановлення та запуск

### 1. Клонування репозиторію
```bash
git clone <your-repo-url>
cd homework-lesson-3
```

### 2. Створення віртуального середовища
Обов'язково використовуйте віртуальне середовище для ізоляції залежностей:
```bash
python -m venv venv
source venv/bin/activate  # Для MacOS/Linux
# або
# venv\Scripts\activate  # Для Windows
```

### 3. Встановлення залежностей
Встановіть всі необхідні бібліотеки:
```bash
pip install -r requirements.txt
```

### 4. Налаштування змінних середовища (.env)
Створіть файл `.env` у кореневій директорії проєкту:
```bash
touch .env
```
Відкрийте файл `.env` та додайте ваш API-ключ для доступу до сервісу Gemini API:
```env
GEMINI_API_KEY="AIzaSyYourApiKeyHere..."
```
*(Ваш `.env` вже додано до `.gitignore`, тож не хвилюйтеся про його випадкове потрапляння в репозиторій).*

### 5. Запуск Агента
Запустіть інтерактивний цикл (REPL) для спілкування з агентом:
```bash
python main.py
```

Коли у консолі з'явиться привітання `Research Agent (type 'exit' to quit)` та `You: `, можете починати ставити завдання українською мовою.

## 📁 Структура проєкту

- `main.py` — Точка входу, містить REPL цикл з красивим форматуванням виводу у консоль.
- `agent.py` — Оголошення та налаштування ReAct агента, пам'яті (checkpointer) та LLM.
- `tools.py` — Імплементація інструментів агента (`@tool`), логіка Context Engineering та обробка виключень.
- `config.py` — Налаштування проекту (Pydantic Settings) та системний промпт.
- `example_output/` — Директорія для згенерованих агентом Markdown-звітів.
