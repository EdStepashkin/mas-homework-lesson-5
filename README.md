
# Research Agent з RAG-системою 🤖📚

Research Agent — це інтелектуальний AI-агент, створений на базі ReAct архітектури (Reasoning and Acting) з використанням **LangGraph** та **LangChain**. В цій версії агент посилений **RAG-системою** (Retrieval-Augmented Generation), що дозволяє йому автономно шукати інформацію як у вашій локальній базі документів (PDF), так і в інтернеті, а також генерувати структуровані звіти у форматі Markdown.

## 🌟 Основні можливості

- **Локальна база знань (RAG)**: Агент спочатку шукає відповіді у локальній векторній базі даних (FAISS), заповненій вашими PDF-документами.
- **Гібридний пошук з Реренкінгом (Hybrid Search + Reranking)**: Поєднує семантичний векторний пошук з лексичним пошуком BM25. Результати фільтруються крос-енкодером `BAAI/bge-reranker-base` для максимальної релевантності.
- **Автономний пошук в інтернеті**: Якщо локальних даних не вистачає, агент звертається до DuckDuckGo для пошуку в інтернеті.
- **Глибокий аналіз сторінок**: Аналізує тексти статей за допомогою веб-скрейпінгу.
- **Context Engineering**: Інтелектуальне обрізання занадто довгих текстів, щоб не переповнювати контекстне вікно моделі.
- **Генерація структурованих звітів**: Уся зібрана інформація оформлюється і самостійно записується агентом у готові `.md` звіти.
- **Підтримка пам'яті (Memory)**: Використовує `MemorySaver` з LangGraph для збереження контексту розмови.

## 🛠 Технологічний стек

- **Основна LLM**: Google Gemini (`gemini-2.5-flash`), ініціалізована через `ChatGoogleGenerativeAI`.
- **RAG-пайплайн**: `FAISS` (Vector DB), `OpenAIEmbeddings` (`text-embedding-3-small`), `BM25Retriever`, `HuggingFaceCrossEncoder` (`BAAI/bge-reranker-base`), `EnsembleRetriever`.
- **Фреймворк**: `langgraph.prebuilt.create_react_agent`
- **Інструменти (Tools)**:
  - `knowledge_search`: знаходження інформації у локальних PDF-документах (використовується в першу чергу).
  - `web_search`: знаходження списку джерел в інтернеті.
  - `read_url`: детальне вичитування інформації за веб-посиланням.
  - `write_report`: запис результатів у директорію `example_output/`.
  - `get_current_time`: отримання поточного часу.
  - `read_local_file`: читання локальних файлів.
- **Конфігурація**: Pydantic `BaseSettings` (`config.py`).

## 🚀 Встановлення та запуск

### 1. Клонування репозиторію
```bash
git clone <your-repo-url>
cd homework-lesson-5
```

### 2. Створення віртуального середовища
Обов'язково використовуйте віртуальне середовище для ізоляції залежностей:
```bash
python -m venv venv
source venv/bin/activate  # Для MacOS/Linux
# або
# venv\Scripts\activate  # Для Windows
```

### 3. Встановлення залежностей
Встановіть всі необхідні бібліотеки:
```bash
pip install -r requirements.txt
```

### 4. Налаштування змінних середовища (.env)
Створіть файл `.env` у кореневій директорії проєкту:
```bash
touch .env
```
Відкрийте файл `.env` та додайте ваші API-ключі:
```env
GEMINI_API_KEY="AIzaSyYourApiKeyHere..."
OPENAI_API_KEY="sk-proj-YourOpenAiKey..."
```
*(Ваш `.env` вже додано до `.gitignore`).*

### 5. Індексація локальних документів (Ingestion)
Перед роботою агента необхідно завантажити знання в базу даних:
1. Помістіть будь-які `PDF` документи в папку `data/`.
2. Запустіть скрипт ідексації:
```bash
python ingest.py
```
Скрипт поріже документи на чанки, згенерує вектори через `OpenAIEmbeddings` і збереже індекси FAISS та BM25 у папку `index/`.

### 6. Запуск Агента
Запустіть інтерактивний цикл (REPL) для спілкування з агентом:
```bash
python main.py
```

Коли у консолі з'явиться привітання `Research Agent (type 'exit' to quit)` та `You: `, можете починати ставити завдання українською мовою.

## 📁 Структура проєкту

- `main.py` — Точка входу, містить REPL цикл агента.
- `ingest.py` — Скрипт завантаження, обробки PDF та побудови гібридного FAISS+BM25 індексу.
- `retriever.py` — Налаштування гібридного пошуку (EnsembleRetriever) та реренкера (CrossEncoder).
- `agent.py` — Ініціалізація бази агента, підключення пам'яті та рушія LangGraph.
- `tools.py` — Імплементація інструментів агента (`knowledge_search`, `web_search`, `write_report`).
- `config.py` — Налаштування (Pydantic Settings) та жорсткий системний промпт агента.
- `data/` — Директорія для ваших вхідних PDF-файлів.
- `index/` — Згенеровані векторні бази даних знань (не додаються в Git).
- `example_output/` — Директорія для згенерованих Markdown-звітів.
